## 1. 各インフォマティクスの役割

このページでは、研究活動を「材料」「計測」「プロセス」「物理」の4つのレイヤーに分けて整理するために導入した  
4つのインフォマティクスの役割と関係性を、簡潔にまとめる。

- 材料インフォマティクス：  
  「どんな材料が良いか？」を、組成・構造・物性のデータから探る。
- 計測インフォマティクス：  
  「何が、どのように測れているか？」を、スペクトル・画像・時系列データから抽出する。
- プロセスインフォマティクス：  
  「どう作れば良いか？」を、プロセス条件と品質・特性の関係から設計する。
- 物理インフォマティクス：  
  「なぜそうなるのか？」を、物理モデルとデータを統合して説明・予測する。

### 材料インフォマティクス（Materials Informatics）

- 対象：
  - 組成、結晶構造、ミクロ構造、物性データ
- 主なタスク：
  - ディスクリプタ設計（組成・構造の数値表現）
  - 物性予測モデル（回帰・分類）
  - ベイズ最適化などによる「有望材料候補」の探索
- 立ち位置：
  - 材料空間を俯瞰し、「どのあたりに狙い目があるか」を示すレイヤー。

### 計測インフォマティクス（Measurement Informatics）

- 対象：
  - スペクトル（XAFS/XPS/XMCD…）、顕微鏡画像、MBN などの時系列信号
  - 計測条件・メタデータ
- 主なタスク：
  - ノイズ除去・ベースライン補正・正規化などの前処理
  - 特徴抽出（ピーク情報、主成分、クラスタリングなど）
  - 自律実験やオンライン解析へのフィードバック
- 立ち位置：
  - 「計る」側の情報量を最大化し、材料・物理・プロセス側に渡すレイヤー。

### プロセスインフォマティクス（Process Informatics）

- 対象：
  - 熱処理条件、成膜条件、加工条件、インラインログ（温度・圧力・電力など）
  - 出力としての品質・特性・歩留まり
- 主なタスク：
  - プロセス条件 → 構造・特性のモデル化
  - ベイズ最適化・DoE によるレシピ探索
  - プロセス監視・異常検知
- 立ち位置：
  - 「どう作れば良いか？」を数理モデルとして整理し、現場のレシピに落とし込むレイヤー。

### 物理インフォマティクス（Physical / Physics Informatics）

- 対象：
  - DFT・MD・フェーズフィールド・マイクロ磁気・FEM などの物理モデル
  - それらの出力と実験データ
- 主なタスク：
  - 物理モデルとデータ駆動モデルの統合（Physics-informed ML / PINNs など）
  - パラメータ同定・逆問題（未知の物性値・モデルパラメータ推定）
  - マルチスケール連成とサロゲートモデル構築
- 立ち位置：
  - 4つのインフォマティクスを貫く「物理的バックボーン」を与えるレイヤー。


## 2. 4つのインフォマティクスの関係性

ざっくりとした関係は、次のように整理できる：

- 材料インフォマティクス  
  ↔（材料の候補・設計指針）↔  
  物理インフォマティクス（物理法則・シミュレーション）

- 計測インフォマティクス  
  ↔（特徴量・観測データ）↔  
  物理インフォマティクス（モデル検証・パラメータ同定）

- プロセスインフォマティクス  
  ↔（プロセス条件・構造変化）↔  
  物理インフォマティクス（プロセス–構造–特性の因果構造）

さらに、

- 材料インフォマティクス  
  ↔ 計測インフォマティクス  
  ↔ プロセスインフォマティクス  

という形で、  
「どんな材料を」「どう測って」「どう作るか」を  
一貫したデータ構造でつなぐ。


## 3. 研究フローにおける使い分けイメージ

例として、新しい軟磁性材料を開発する場合：

1. 材料インフォマティクス  
   - 組成・構造・物性データから、  
     「高磁化・低損失になりうる組成領域」をデータ駆動で絞り込む。

2. 物理インフォマティクス  
   - DFT・MD・マイクロ磁気などで、  
     磁気モーメント・異方性・損失メカニズムを解析し、  
     候補材料の「物理的な理由」を確認する。

3. プロセスインフォマティクス  
   - 熱処理・成膜条件を変えながら、  
     磁気特性・損失・組織の関係をモデル化し、  
     最適なプロセスレシピを探索する。

4. 計測インフォマティクス  
   - XAFS/XMCD/XRD、MBN、MOKE 画像などから、  
     ノイズを抑えた特徴量を抽出し、  
     上記 3 つのモデルの検証・改善にフィードバックする。


## 4. まとめ

- 各インフォマティクスの詳細は、個別ページ（材料・計測・プロセス・物理）に分けて整理する。
- 本ページは、
  - 「どの話題がどのインフォマティクスに属するか？」
  - 「どのレイヤーを今見ているのか？」
  を確認する「ハブ」として位置づける。

研究ノートや授業資料を整理する際にも、  
- これは材料インフォマティクスの話か？  
- これは計測・プロセス・物理のどこか？  
とタグ付けすることで、  
後から全体像を掴みやすくすることを意図している。
